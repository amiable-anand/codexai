# Project Summary
The CodexAI project is an advanced system for codebase documentation and management, aimed at enhancing software development efficiency. It features a command-line interface (CLI) for uploading code to Azure, an Azure Functions backend for processing code, and a React frontend for user interaction. The project focuses on streamlining documentation processes, improving code understanding, and facilitating project management through intelligent integration of various Azure services.

# Project Module Description
## 1. CLI Tool (`/workspace/cli/`)
- **codexai_upload.py**: A Python script for uploading codebases to Azure Blob Storage.
- **setup.sh**: An automatic setup script to prepare the CLI environment.

## 2. Azure Functions Backend (`/workspace/backend/`)
- **api_handler.py**: Provides HTTP-triggered API endpoints.
- **code_chunker.py**: Breaks down code into manageable chunks using AST.
- **cosmos_service.py**: Manages metadata in Cosmos DB.
- **embedding_service.py**: Generates embeddings using Azure OpenAI.
- **ingestion_handler.py**: Handles blob-triggered ingestion pipelines.
- **rag_handler.py**: Implements the RAG inference engine with vector search capabilities.
- **requirements.txt**: Lists all necessary Python dependencies.
- **search_service.py**: Integrates with Azure Cognitive Search for code retrieval.

## 3. React Frontend (`/workspace/frontend/`)
- **index.html**: The main HTML file for the React application.
- **App.tsx**: The root component of the React application.
- **components/**: Contains various components for project browsing and documentation viewing.
  - **DocumentationViewer.tsx**: Displays Markdown documentation.
  - **FileExplorer.tsx**: Allows navigation through the file structure.
  - **ProjectList.tsx**: Lists projects for user selection.
- **package.json**: Defines project dependencies.
- **vite.config.ts**: Configuration for Vite, the build tool used.

## 4. Configuration & Documentation
- **README.md**: Comprehensive setup guide including security warnings.
- **docs/**: Contains various documentation files, including setup guides and design diagrams.

# Usage
1. **Set up Azure resources**: Follow the instructions in `docs/azure_setup_guide.md`.
2. **Configure environment variables**: Copy `.env.template` to `.env` and fill in your Azure credentials.
3. **Install dependencies**: Run the setup scripts for CLI, backend, and frontend.
4. **Deploy to Azure**: Deploy the backend functions and frontend app.
5. **Upload your first codebase**: Use the CLI tool to upload and generate documentation.
